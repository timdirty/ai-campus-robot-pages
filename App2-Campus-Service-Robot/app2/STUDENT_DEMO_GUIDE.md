# 學生操作與報告講解稿：校園服務機器人 App

## 一句話介紹

我們這組做的是「校園服務機器人中控台」。它可以把福利社配送、教學協助、清潔廣播、校園巡邏和安全任務整合在同一個 App 裡，讓機器人不只是單一功能，而是校園裡的任務助手。

## 學生操作入口

- 公開展示網址：`https://timdirty.github.io/115-campus-ai-demo/app2/`
- 本機開發版：執行 `npm run dev` 後開啟 Vite 顯示的網址。
- 沒有 Arduino 時，任務仍會建立，指令 log 會顯示未連線或 fallback。

## 上台分工建議

- 第一位同學：介紹校園痛點與中控台概念。
- 第二位同學：操作配送下單、庫存扣除、任務建立。
- 第三位同學：操作追蹤送達、教學提醒或生活服務。
- 第四位同學：展示派遣、安全封鎖、報表與硬體指令 log。

## 3 分鐘講解稿

大家好，我們是校園服務機器人小組。我們觀察到學校裡有很多重複性工作，例如福利社配送、清潔提醒、廣播、巡邏、上課點名和安全通報。如果每一件事都分開做，老師和行政人員會很忙，所以我們設計一個校園機器人中控台。

首先在首頁可以看到今天的任務鏈、機器人狀態和 UNO R4 指令 log。這代表我們每個操作不只是畫面變化，而是會同步影響任務、庫存、機器人狀態和硬體指令。

接著我們進入配送頁，下訂一筆福利社商品。送出後，系統會扣庫存、建立訂單、指派機器人，並把配送指令送到硬體橋接流程。如果庫存不足，系統只會記錄錯誤，不會誤派機器人。

再到追蹤頁，我們可以把配送任務標記為送達，機器人會回到待命。接著可以展示教學頁，例如點名、學生提問回覆或分心提醒，代表機器人也可以協助課堂。

最後我們到派遣和報表頁。派遣頁可以發送巡邏、廣播或安全任務；報表頁可以讓評審看到今天完成了哪些服務。未來接上 Arduino 後，這些任務會變成實體機器人的燈號、移動或廣播動作。

## 操作步驟卡

1. 開啟首頁，看任務鏈、機器人狀態與指令 log。
2. 到配送頁選商品並下單。
3. 確認庫存、訂單、任務和機器人狀態同步改變。
4. 到追蹤頁完成送達。
5. 到教學頁完成點名或處理提醒。
6. 到生活服務頁安排清潔或廣播。
7. 到派遣頁發送巡邏或安全任務。
8. 回首頁看任務鏈與 UNO R4 指令 log。
9. 最後開報表或重置展示資料。

## 評審可能會問

### 這是不是只有按鈕切頁？

不是。主要操作會同步改變庫存、訂單、任務、機器人狀態、系統 log 和硬體指令紀錄。

### 如果商品沒庫存會怎樣？

系統會阻止錯誤任務，只寫入錯誤 log，不會派遣機器人，也不會送出硬體指令。

### 沒有 Arduino 可以展示嗎？

可以。沒有 Arduino 時仍可完成整個任務流程，指令 log 會顯示待送或未連線；接上 UNO R4 後同一批任務會走 Serial。

## 後續機器人連動計畫

1. 將 `DELIVERY_START` 對應到機器人出發燈號或移動動作。
2. 將 `DELIVERY_DONE` 對應到抵達提示或蜂鳴聲。
3. 將 `BROADCAST_START`、`PATROL_START`、`SAFETY_LOCKDOWN` 對應到 LED Matrix 圖示。
4. 加入安全停止與手動暫停，避免實機展示時碰撞。
5. 保留 App 目前 fallback，讓比賽現場即使硬體斷線也能完成軟體展示。
