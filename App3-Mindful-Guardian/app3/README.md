# AI 校園心靈守護者

本專案是「AI 學校情緒檢測裝置」的產品級本機示範。第一屏是亮色專業儀表板：AI 主動巡查圖書館、穿堂、教室、體育館與操場等區域，用 2.5D 校園平面顯示風險、聲量、提醒數與機器人任務；Firebase 與 Gemini 都是未來可接選項，目前不用登入、不用金鑰也能完整展示。

感知中心可啟用瀏覽器麥克風做本機即時運算，只計算音量指標與波動，不儲存原始聲音、不轉文字、不上傳雲端；運算後的匿名訊號可建立關懷提醒，作為老師巡查與觀察的輔助資料。

AI 主動巡查會融合學生自填、聲量波動、節點狀態與既有預警，不等學生求助就產生「需要觀察」或「優先關懷」提醒。它只做風險提示與老師巡查建議，不宣稱心理診斷。

預警、聲量感知、自我照護、聊天支持、節點與紀錄都收進右側或底部抽屜。主畫面只保留評審最需要看懂的主線：看校園地圖、點選風險區、派遣機器人介入，並同步留下機器人任務與硬體提示紀錄。

所有機器人互動都有前端回饋：按鈕狀態、地圖脈衝、任務進度、硬體備援紀錄與提示訊息會同步更新。還沒接上實體機器人時也能完整展示流程；接上 Arduino 橋接服務後，同一批操作會改送到實體裝置。

硬體提示路徑使用 App 3 專屬本機 Arduino 橋接服務，預設 `http://localhost:3203/api/robot/command`，可用 `VITE_ARDUINO_BRIDGE_URL` 改成其他橋接網址；預警處理、佈署關懷與節點重新連線都會發出對應 UNO R4 指令。沒有插 Arduino 時只顯示展示備援狀態，插上並上傳韌體後，同一批指令會送到序列埠。

紀錄抽屜會保留最近硬體提示與機器人任務，評審可以看到「已送」或「備援」狀態，不必只依賴瞬間提示。

## 本機啟動

```zsh
npm install
npm run dev
```

開啟 Vite 顯示的網址。

## 驗證

```zsh
npm run check
```

`check` 會執行本機狀態與 AI 測試、TypeScript 檢查與正式建置。

## 展示資料

資料保存在瀏覽器：

```text
mindful-guardian:v1
```

App 右上角「匯出」可下載目前示範資料檔，「匯入」可載入另一台電腦或前一輪展示資料並先做安全修復，「重置」可回到初始示範資料。所有學生資料都是匿名示範代號，不存放真學生個資。

## 10 秒上手示範

學生可直接照著操作與報告的版本請看 `STUDENT_DEMO_GUIDE.md`。

1. 第一屏看 2.5D 校園地圖，說明 AI 正在主動巡查匿名場域訊號。
2. 指出最高風險區域的風險、聲量與提醒數。
3. 點選中高風險區域，派遣機器人介入。
4. 打開「預警」抽屜，看老師低壓確認與關懷處置。
5. 打開「感知」抽屜，展示本機聲量運算與 AI 主動巡查。
6. 打開「紀錄」抽屜，確認硬體提示與機器人任務已留下證據。

## 現場故障備案

- 瀏覽器本機資料壞掉或空資料：App 會恢復初始匿名示範資料。
- 無 Firebase/Gemini：維持本機優先流程與本機回覆，不影響展示。
- 無麥克風權限：可使用示範高聲量訊號，不影響展示。
- 無 Arduino：關懷流程照常完成，硬體提示會以橋接備援提示和總覽紀錄告知；插上 UNO R4 後，同一操作會改送到序列埠。
- 不使用真學生資料：所有欄位維持匿名代號，說法只限「關懷提醒」與「輔導建議」。
- 隱私、匯出、匯入、重置與匿名投稿都有即時回饋，方便評審確認操作成功。

## 評分亮點

- 第一屏是亮色產品級校園指揮中心，不再是多頁工具台；評審能先看到「AI 主動守護 → 發現風險 → 派遣機器人」。
- 預警處理、自我照護、聊天支持與節點監控都保留在抽屜面板，形成完整關懷閉環。
- 學生自填、環境聲量與節點資料形成多來源訊號；AI 主動巡查會融合這些訊號產生關懷提醒。
- 聲音只保留運算後指標，不保存錄音。
- 硬體提示紀錄會保存在瀏覽器本機資料與匯出檔中，可證明關懷/節點指令有送到橋接服務。
- 機器人派遣任務會保存在瀏覽器本機資料與匯出檔中，可證明高風險空間已觸發介入流程。
- 語氣聚焦匿名、非診斷、支持與轉介，不宣稱醫療或心理診斷能力。
- 巡邏底盤可收折遙控列：底部常駐 D-pad 控制欄，手動操控前後左右、速度、緊急停車；收折時僅顯示連線狀態，韌體看門狗 3 秒保護。
- 主要 UI 元件已拆到 `src/components/guardianUi.tsx`，狀態恢復集中在 `src/state/guardianState.ts`。
- `STUDENT_DEMO_GUIDE.md` 已整理學生上台分工、逐字講稿、操作步驟卡與後續機器人連動說法。

## 後續正式整合

- Firebase：可作為正式雲端同步層，但必須保留本機備援流程。
- Gemini：建議未來改由後端代理呼叫，不在前端暴露金鑰。
- Arduino UNO R4 WiFi：目前已可送並記錄 `ALERT_SIGNAL`、`CARE_DEPLOYED`、`NODE_RESTART` 到本機橋接服務；下一步只需要上傳韌體並驗證實體 LED、矩陣或伺服動作。
