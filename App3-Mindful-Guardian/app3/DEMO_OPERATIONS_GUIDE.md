## 一、系統架構速覽

```
電腦（老師/主控）               iPad（機器人上）
┌──────────────────────┐        ┌───────────────────┐
│  瀏覽器 :3000         │        │  robot-display    │
│  主控儀表板（App3）   │        │  robot_app3.jsx   │
│       ↕               │        │       ↑            │
│  Bridge Server :3203  │──WiFi──│  WebSocket /display│
│  （橋接 Arduino/iPad）│        └───────────────────┘
│       ↕               │
│  Arduino UNO R4       │   ← 有硬體才需要，沒有也能完整示範
└──────────────────────┘
```

**最低需求（無硬體也能完整示範）：**
- MacBook / Windows 電腦一台
- Node.js 18+（`node --version` 確認）
- 現代瀏覽器（Chrome / Edge 建議）

**完整展示（加 iPad 同步）：**
- 電腦 + iPad 同一 WiFi
- iPad 開瀏覽器（Safari / Chrome）

## 二、快速啟動（2 分鐘）

### A. 純前端模式（最簡單，無需伺服器）

```bash
cd "AI校園心靈守護者"
npm install          # 第一次才需要
npm run dev:web      # 只跑前端
```

瀏覽器開 → `http://localhost:3000`

> ✅ 適合：快速練習，不需要 iPad 同步、不需要 Arduino

### B. 完整模式（前端 + 橋接伺服器 + iPad 同步）

```bash
cd "AI校園心靈守護者"
npm install          # 第一次才需要
```

終端機會出現兩行確認訊息：

## 三、iPad 機器人顯示連線

### 步驟 1：確認同一 WiFi

電腦和 iPad 必須連到**同一個** WiFi（例如競賽場地的 WiFi 或手機熱點）。

查電腦 LAN IP：
```bash
# macOS
ipconfig getifaddr en0    # WiFi
# 或
ifconfig | grep "inet " | grep -v 127
```

記下 IP，例如 `192.168.0.34`

### 步驟 2：在主控儀表板取得 QR Code

1. 打開主控儀表板 `http://localhost:3000`
2. 點底部 **機器人** Tab
3. 找到「機器人顯示 · ROBOT FACE」卡片，點開展開
4. 點 **「產生 QR 碼」** 按鈕
5. 出現 QR Code 圖片

> 若橋接伺服器未啟動（純前端模式），QR 無法產生。改用手動輸入。

### 步驟 3：iPad 掃碼或手動輸入

**方法 A：掃 QR Code（建議）**
- iPad 開相機 App，掃 QR Code → Safari 自動開啟

**方法 B：手動輸入 URL**
- 格式：`http://[電腦LAN IP]:3000/robot-display.html?bridge=[電腦LAN IP]:3203`
- 範例：`http://192.168.0.34:3000/robot-display.html?bridge=192.168.0.34:3203`

### 步驟 4：確認連線成功

iPad 畫面出現機器人臉 + 顯示「**SYNCED**」綠色標籤 = 連線成功。

主控儀表板「機器人顯示」區塊顯示 `[iPad 已連線]` = 成功。

## 四、App 3 功能模組說明

### 主畫面（第一屏）

| 區塊 | 功能說明 |
|---|---|
| 2.5D 校園地圖 | 點選任一空間區域可查看風險分數、聲量、提醒數 |
| 最高風險區橫幅 | 顯示當前最需關注的校園空間，點可直接派遣機器人 |
| 風險指數環圖 | 即時融合分數視覺化（0-100） |
| 自動示範按鈕（右上角，桌面版） | 5 秒內自動跑完 3 步完整流程 |
| 匯出 / 匯入 / 重置（右上角圖示） | 資料管理 |

### 底部 4 個 Tab

```
預警    感知    照護    機器人
 🔔      📡     💚       🤖
```

#### 預警 Tab
- 查看所有 AI 生成的關懷提醒
- 點選一筆提醒 → 展開「處置清單」（導師確認 / 安排談話 / 轉介輔導室）
- 勾選後按「佈署關懷」→ 建立支持方案、送硬體 `CARE_DEPLOYED` 訊號

#### 感知 Tab
- **AI 融合分析**：4 訊號分數條（心情 0-2 / 聲量 0-3 / 節點 0-3 / 未結提醒 0-2）
- **聲量監控**：點「示範高聲量」注入示範數據；或啟用麥克風做真實採樣
- **校園聲量趨勢圖**：SVG 折線圖，顯示 ↑↓→ 趨勢箭頭
- **主動判斷**：按「由多來源訊號建立提醒」→ AI 自動建立預警並記錄

#### 照護 Tab
- **心情簽到**：4 種匿名心情（開心 / 還可以 / 有點累 / 有點擔心）
- **心靈森林**：匿名貼文牆，貼一篇心情貼文
- **安全空間聊天**：本機 AI 回覆（或可選 Gemini API）
- **情緒字體**：輸入一段話 → AI 分析情緒色彩

#### 機器人 Tab
- **校園空間派遣**：選區域 → 派遣機器人
- **快速指令**：警示信號 / 部署關懷 / 開始巡查 / 暫停 / 繼續 / 鎖定
- **巡邏控制台**（底部可收折）：D-pad 前後左右 + 速度 + 緊急停車
- **節點監控**：各感測節點狀態 + 重新連線
- **硬體提示紀錄**：完整操作留存紀錄（可追蹤證據）
- **機器人顯示同步**：QR Code 產生 + iPad 情緒手動覆寫

## 五、完整 Demo 操作步驟（5 分鐘）

### 開場前準備

```
□ npm run dev 已啟動，瀏覽器開著
□ 若有 iPad：掃碼連線確認 SYNCED
□ 資料：點右上角「重置」清空，從乾淨狀態開始
```

---

### 第一屏說明（30 秒）

> 「我們做的不是診斷工具，而是匿名主動巡查系統。
> 這張地圖每個區域都有 AI 計算的風險分數，融合了心情、聲量、節點、提醒四種訊號。」

操作：指出地圖上最高風險的紅色區域，說明「風險 XX / 100」。

---

### 自動示範（選用，最快 5 秒完整示範）

點右上角 **「自動示範」** 按鈕：

1. ① 0.8 秒後：AI 建立預警 → 自動切換到「預警 Tab」
2. ② 2.4 秒後：聲量訊號記錄 → 切換到「感知 Tab」
3. ③ 4.2 秒後：派遣機器人 → 切換到「機器人 Tab」

---

### 手動完整流程（建議）

**步驟 1：校園地圖 + 機器人派遣（60 秒）**

```
① 點校園地圖上的「穿堂」（或任一中高風險區）
② 側邊卡片出現：點「派遣機器人」
③ 機器人動畫 + Toast「機器人派遣中…」
④ 若有 iPad：觀察機器人臉部切換到「專注」表情
```

**步驟 2：預警處理（60 秒）**

```
① 點底部「預警」Tab
② 點一筆「AI 主動巡查」提醒，展開細節
③ 勾選「導師已確認」→「安排關懷談話」
④ 點「佈署關懷」→ 建立支持方案
⑤ 系統顯示「硬體提示：CARE_DEPLOYED」
```

**步驟 3：感知中心（45 秒）**

```
① 點底部「感知」Tab
② 指出「AI 融合分析」4 條訊號（心情/聲量/節點/提醒）
③ 點「示範高聲量」→ 折線圖出現上升趨勢
④ 點「由多來源訊號建立提醒」→ 新增一筆預警
```

**步驟 4：照護功能（60 秒）**

```
① 點底部「照護」Tab
② 點「有點擔心」做心情簽到 → Toast 確認
③ 在心靈森林輸入一則貼文，點「發布」
④ 在聊天框輸入「我今天有點緊張」→ AI 回覆
```

**步驟 5：機器人 + 紀錄（45 秒）**

```
① 點底部「機器人」Tab
② 點「CARE_DEPLOYED 部署關懷」快速指令
③ 下滑到「硬體提示紀錄」→ 展示操作留存證據
④ 展示「機器人顯示」區塊，點開 QR Code 說明 iPad 同步
```

**結尾（15 秒）**

```
點右上角「匯出」→ 下載 JSON 檔
說：「整個流程都有可追蹤的記錄，可以匯出給輔導老師參考。」
```

## 八、疑難排解

### 橋接伺服器連不上

```bash
# 確認 port 是否被佔用
lsof -i :3203
# 若有舊程序，重新啟動即可（bridge 有自動 kill 舊 pid）
npm run dev
```

### iPad 無法連線

```
□ 確認電腦和 iPad 同一 WiFi（不是 USB Tethering）
□ macOS：系統設定 → 防火牆 → 確認允許傳入連線
□ 手動輸入 URL：http://[電腦IP]:3000/robot-display.html?bridge=[電腦IP]:3203
□ 用 en0（WiFi）的 IP，不要用 en1（Thunderbolt）
```

### 感知 Tab 出現「趨勢資料收集中」

麥克風尚未採樣，正常現象。點「示範高聲量」立即注入示範數據。

### 點「自動示範」無反應

示範仍在進行中（共 5.2 秒），請稍等。若 Toast 出現「示範進行中」代表防抖保護啟動，等一下再點。

### 資料看起來異常 / 想重置

點右上角「重置」圖示（循環箭頭）→ 確認清除 → 全部恢復初始示範資料。

### 瀏覽器 localStorage 損毀（極少見）

```javascript
// 在瀏覽器 Console 執行
localStorage.removeItem('guardian-state:v3');
localStorage.removeItem('guardian-sound-trend:v1');
location.reload();
```
